# Public-Health-Disease-Surveillance- Configuration Testing and Virtual Machine Project

This project focused on building the public health Architecture through an interoperability structure that helps public health analysts exchange information between electronic records across different health facilities and obtain information that would help population health

 This involves developing a  virtual machine environment according to the illustration below. Virtual Machines (VMs) are a terrific way to test various network architectures and developments in a safe environment before adding them to production environments, such as on-premises hardware or cloud-based solutions. So, four different virtual machines were created to simulate the different hospital facilities, and additionally, a centralised health information exchange virtual machine was created. Each virtual machine for the hospital has its own IP address, enabling connections to each facility and facilitating exchange. This was done using the Ubuntu Linux system. Each of these virtual machines was assessed to check if its operating system would be compatible with open EMR, where patient information are collected or recorded within the hospital system.

## The outcome of this project:

This was to simply portray and simulate how the exchange of health information is done in a real-world scenario.
